## 已决策事项

| # | 问题 | 决策 | 理由 |
|---|------|------|------|
| 1 | 技术栈选择 | Vite + Vue 3 + TypeScript | 兼顾复杂度与可维护性，组件化架构便于增删页面 |
| 2 | 路由模式 | Hash 路由 (`createWebHashHistory`) | GitHub Pages 不支持 SPA fallback，hash 路由无 404 问题 |
| 3 | 内容管理方式 | TypeScript 数据文件 (`src/data/*.ts`) | 你负责用我更新，TS 数据文件比 JSON 多类型检查，改起来也直观 |
| 4 | 导航方案 | 桌面端固定左侧栏 + 移动端顶部汉堡菜单 | 符合你的要求，响应式适配 |
| 5 | 配色方案 | 深蓝 (#034484) + 金色 (#C9A84C) | 从队徽提取的主色调，暖白底 (#F8F6F1) 配衬线字体 |
| 6 | 字体选择 | Noto Serif SC（中文衬线）+ Inter（英文无衬线） | 免费可商用，兼顾古朴感与现代阅读体验 |
| 7 | 首页风格 | 极简 Hero + 快速导航卡片网格 | 你说不喜欢首页堆太多内容，二级页面承载具体信息 |
| 8 | 卡片 UI 风格 | 浮动卡片（阴影 + hover 上浮效果） | 满足你想要的"悬浮页立体设计" |
| 9 | 视频方案 | B站内嵌为主，预留外部视频床接口 | B站免费且不限流量，未上传社交媒体的视频用其他视频床 |
| 10 | 搜索功能 | 前端简单搜索（标题+描述匹配） | 数字档案库和口述史页面已实现，数据量增大后可升级 |
| 11 | 访问统计 | 未实现 | 你要求仅后端显示用户不可见，当前纯静态无法实现，后续可加 |
| 12 | 部署方式 | GitHub Actions 自动构建 + Pages 部署 | 免费，push 即自动更新 |
| 13 | 构建路径 | base: `/jinmen-xunmai/` | 匹配 GitHub 仓库名 |
| 14 | 背景装饰 | 宣纸底纹（CSS颗粒渐变）+ 金线装饰 + 卡片角标 | 提升视觉丰富度，不喧宾夺主 |
| 15 | 页面动画 | 卡片滚动进入淡入上浮 + 队徽呼吸光晕 + 导航流光 | 用 Intersection Observer 实现，仅触发一次 |
| 16 | 移动端遮罩 | 半透明黑(0.35) + backdrop-filter blur(2px) | 比纯黑半透明更柔和 |
| 17 | 移动端滚动锁定 | 侧栏打开时 body.sidebar-open { overflow: hidden } | 防止背景页面滑动 |
| 18 | Logo防遮挡 | 移动端侧栏添加 padding-top: var(--topbar-height) | 避免侧栏内容被顶栏遮盖 |

## 后续开发建议（按优先级）

1. **填写真实内容** — 成员信息、实践动态、档案资料（最立竿见影的"去素"手段）
2. **接入 B站嵌入** — 拿到 BV 号后替换 `media.ts` 中的占位
3. **更多图片素材** — 实际照片、扫描件填入后视觉丰富度大幅提升
4. **PWA 支持** — 可以配置成可安装的渐进式 Web 应用
5. **访问统计** — 用 Vercel Analytics 或 百度统计（免费）
6. **图片懒加载** — 档案库图片增多后很有必要
